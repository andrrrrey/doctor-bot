# Doctor Bot — Платформа опроса по боли в спине

Медицинская консультационная платформа для диагностики боли в спине, созданная на основе 17-летнего клинического опыта доктора Антона Эпифанова (невролог, специалист по заболеваниям позвоночника).

Система принимает пациента через интерактивный опросник, автоматически формирует диагноз и передаёт данные в клинику по электронной почте и в CRM Bitrix24.

## Обзор архитектуры

Проект состоит из трёх независимых частей:

```
doctor-bot/
├── server/        # Backend на Express.js + TypeScript
├── client/        # Frontend-скрипты для основного опросника
├── telegram/      # Telegram-бот
└── public/        # Статические файлы (виджет, админ-панель)
```

Все три части общаются через REST API сервера. Telegram-бот и виджет — самостоятельные клиенты, которые вызывают те же публичные эндпоинты, что и основной сайт.

## Функционал

### Публичный интерфейс

**Основной опросник (`/`)**

Пошаговый опросник по болям в спине. Вопросы строятся динамически: ответ на предыдущий вопрос определяет, какие вопросы будут заданы следующими.

Поддерживаемые типы вопросов:
- `radio` — выбор одного варианта ответа
- `checkbox` — выбор нескольких вариантов
- `number` — числовой ввод (например, возраст)
- `text` — произвольный текст
- `file` — загрузка файла (например, МРТ-снимки)

После прохождения опросника система:
1. Рассчитывает диагноз на основе весовых коэффициентов ответов
2. Выводит HTML-результат с персонализированными рекомендациями
3. Показывает форму для записи на консультацию

**Диагностика**

Алгоритм анализирует симптомы по нескольким осям:
- Тип боли: мышечная, грыжа, артроз, стеноз, воспаление
- Уровень воспаления (умеренное / выраженное)
- Психосоматический компонент (стресс, невроз)
- Интенсивность боли по шкале

**Форма консультации**

Пациент оставляет имя, телефон и город. Система:
- Генерирует PDF с ответами пациента
- Отправляет данные на email клиники
- Создаёт лид в Bitrix24

**Виджет (`/widget`)**

Самодостаточный виджет для встраивания на сторонние сайты. Включает в себя весь опросник и форму консультации. Пациент соглашается с политикой конфиденциальности перед началом.

---

### Telegram-бот

Полный аналог веб-опросника внутри Telegram:

- Запуск по команде `/start`
- Вопросы отображаются как сообщения, варианты ответов — как inline-кнопки
- Числовой ввод (возраст) принимается текстовым сообщением
- Условная логика follow-up-вопросов работает так же, как в веб-версии
- После опросника выводится диагноз
- Пациент может оставить контактные данные для записи на консультацию — бот пересылает их в систему (email + Bitrix24)

### Административная панель

Доступна по адресу `/admin`. Требует авторизации.

#### Роли пользователей

| Роль | Возможности |
|------|------------|
| `superadmin` | Полный доступ ко всем разделам |
| `doctor` | Просмотр заявок и аналитики (без изменения вопросов и настроек) |

#### Управление вопросами *(только superadmin)*

- Создание, редактирование, удаление вопросов
- Настройка типа вопроса, порядка отображения, условий показа
- Настройка вариантов ответов с весовыми коэффициентами для алгоритма диагностики
- Включение/отключение вопросов без удаления

#### Управление пользователями *(только superadmin)*

- Создание новых администраторов (superadmin или doctor)
- Изменение роли и пароля пользователей
- Удаление с защитами: нельзя удалить последнего суперадминистратора и нельзя понизить себя

#### Просмотр заявок

- Список всех заявок с пагинацией
- Фильтры: диапазон дат, статус синхронизации с Bitrix, наличие файла
- Детальный просмотр: данные пациента + все ответы
- Повторная отправка заявок в Bitrix при ошибке синхронизации

#### Аналитика

Воронка конверсии с возможностью фильтрации по датам:

```
Сессий начато → Опрос завершён → Заявок отправлено → Передано в CRM
```

Дополнительно:
- Процент завершения опросника
- Процент конверсии в заявку
- Статистика синхронизации с Bitrix (успешно / ошибки)

#### Версионирование опросника *(только superadmin)*

- Сохранение снапшота текущего состояния опросника
- История версий с указанием количества заявок по каждой версии
- Просмотр любой исторической версии

#### Настройки *(только superadmin)*

Все настройки хранятся в базе данных и редактируются через UI:

- SMTP: хост, порт, логин, пароль, флаг SSL
- Список email-получателей
- URL webhook Bitrix24
- Заголовок лида в Bitrix24

## Установка и запуск

### Переменные окружения

Создайте файл `server/.env` на основе примера ниже:

```env
# Сервер
PORT=3000
NODE_ENV=production

# База данных
DATABASE_URL=postgresql://USER:PASSWORD@localhost:5432/doctor_bot

# JWT
JWT_SECRET=your-very-secret-key-change-this

# Загрузка файлов
UPLOAD_DIR=./uploads
MAX_FILE_SIZE_MB=10

# SMTP (можно переопределить через UI админ-панели)
SMTP_HOST=mail.beget.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your-email@example.com
SMTP_PASS=your-smtp-password
EMAIL_RECIPIENTS=clinic@example.com

# Bitrix24 (можно переопределить через UI админ-панели)
BITRIX_WEBHOOK_URL=https://your-domain.bitrix24.ru/rest/123/your-token/
```

Создайте файл `telegram/.env`:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
API_BASE_URL=http://localhost:3000
```

### База данных

```bash
# Создайте базу данных PostgreSQL
psql -U postgres -c "CREATE DATABASE doctor_bot;"

# Примените миграции
cd server
npm install
npx prisma migrate deploy

# Заполните начальными данными (вопросы и первый superadmin)
npm run db:seed
```

После сидирования будет создан суперадминистратор с логином и паролем из скрипта `seed.ts`. Смените пароль сразу после первого входа.

### Сервер

```bash
cd server
npm install

# Разработка (hot reload)
npm run dev

# Production
npm run build
npm start
```

Сервер будет доступен на `http://localhost:3000`.

### Telegram-бот

```bash
cd telegram
npm install

# Разработка
npm run dev

# Production
npm run build
npm start
```

### Запуск через PM2

В корне проекта находятся два PM2-конфига:

**Запустить сервер и Telegram-бот вместе:**

```bash
pm2 start start.config.js
```

**Запустить только Telegram-бот:**

```bash
pm2 start start-telegram.config.js
```

**Управление процессами:**

```bash
pm2 status          # Статус всех процессов
pm2 logs            # Логи в реальном времени
pm2 restart all     # Перезапустить всё
pm2 save            # Сохранить конфигурацию
pm2 startup         # Автозапуск при старте системы
```

## Интеграции

### Bitrix24

При отправке заявки система автоматически создаёт лид в Bitrix24 через REST API (webhook). Заголовок лида, поля (имя, телефон, город, диагноз) настраиваются через админ-панель. В случае ошибки статус сохраняется в БД, и лид можно повторно отправить вручную из раздела заявок.

### Email / SMTP

После отправки заявки на email клиники приходит письмо с данными пациента и PDF-вложением, содержащим все ответы опросника. SMTP-параметры задаются через переменные окружения или переопределяются в настройках админ-панели.

### Yandex.Metrika / Roistat

В HTML-страницах подключены счётчики Яндекс.Метрики и Roistat для отслеживания событий воронки.
